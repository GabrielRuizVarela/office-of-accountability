Graph Interactivity — Two Approaches

Both deliver the same features, different implementation strategy

A

Incremental Upgrade

Keep react-force-graph-2d. Add features on top of the existing canvas renderer.

Pros

  • No library migration — lower risk
  • Existing code (search, filters, detail panel) stays intact
  • Canvas rendering is fast for large graphs
  • Custom node painting already works — extend it for icons & smart density

Cons

  • Path highlighting and pinning require manual state management
  • No built-in layout algorithms beyond force-directed
  • Hit-testing for hover/click is manual (canvas, not DOM)
B

Migrate to Cytoscape.js

Replace react-force-graph-2d with Cytoscape.js — purpose-built for graph analysis.

Pros

  • Built-in shortest path, BFS, clustering algorithms
  • Native node pinning, compound nodes, layout switching
  • Rich selector engine (CSS-like) for styling by data
  • Export/import graph state trivially (JSON serialization)

Cons

  • Full rewrite of graph components (~5 files)
  • DOM-based rendering can be slower at 500+ nodes (mitigated by canvas extension)
  • Different API — search, filters, detail panel need adaptation

My recommendation: A (Incremental) — You already have a working canvas renderer with good performance. The features you need (expand-in-place, path finding, pinning, smart labels) are all achievable without a full rewrite. A new API endpoint for shortest-path and client-side state management gets you there with less risk. Save the Cytoscape migration for if/when you need advanced graph algorithms at scale.
