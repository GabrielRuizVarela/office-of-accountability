## 5. Technical Improvements

### 5.1 Graph API Performance

- **RELATED_TO scan cost:** 288 generic RELATED_TO edges force full-scan filtering when querying for specific relationship subtypes. Retyping to specific labels enables Neo4j's index-free adjacency.
- **Degree-1 node density:** 48% of nodes being degree-1 means shortest-path queries often dead-end. Improving connectivity directly improves analytical query results.
- **Missing fulltext index:** Creating a fulltext index on `Person.name` and `Organization.name` across the platform graph would unlock name-matching at scale (currently O(n*m) on 2.3M entities).

```cypher
-- Recommended index creation
CREATE FULLTEXT INDEX personNameIndex FOR (n:Person) ON EACH [n.name]
CREATE FULLTEXT INDEX orgNameIndex FOR (n:Organization) ON EACH [n.name]
CREATE FULLTEXT INDEX officerNameIndex FOR (n:CompanyOfficer) ON EACH [n.name]
```
